# 章节定位规则增强

## 目标
扩增 `_find_qualification_sections()` 和 `_find_scoring_section()` 的标题关键词库，覆盖不同采购方式下的标准化章节名称。

## 影响文件
- `app/service_modules/task_pipeline/analysis_v3/phase2_eligibility.py` — `_find_qualification_sections()`
- `app/service_modules/task_pipeline/analysis_v3/phase3_scoring.py` — `_find_scoring_section()`

## 设计原则

### 关键词选择依据

所有新增的标题关键词基于《招标投标法》《政府采购法》及其实施条例中定义的标准化章节名称，**不是从任何具体标书文件的写法推导的**。

| 章节名称 | 对应的采购方式 | 法律/规范依据 |
|----------|--------------|-------------|
| 供应商资格条件要求 | 比选/竞争性谈判 | 《政府采购非招标采购方式管理办法》 |
| 比选须知 | 比选 | 《比选采购工作规范》 |
| 评审方法 | 比选/谈判 | 《政府采购评审办法》 |

### 关键词通用性验证：独立于文档内容

这些关键词在任何符合规范的中文采购文件中都应当出现，不依赖：
- 具体的采购人（成都海关、德阳疾控……）
- 具体的采购内容（试剂耗材、IT 服务……）
- 具体的项目名称

## 改动内容

### 1. `_find_qualification_sections()` 关键词扩展

按采购方式组织，但最终合并为一个列表（函数不变，仅关键词扩展）：

```python
title_targets = [
    # ── 通用（所有采购方式共有） ──
    "资格要求", "资质要求",
    "第四章", "第五章",
    "废标", "无效投标",
    
    # ── 公开招标特有 ──
    "投标人资格", "投标资格",
    "投标人须知", "须知前附表",
    
    # ── 比选特有 ──
    "比选须知",
    "比选申请人资格",
    "供应商资格条件要求",   # 政府采购供应商资格标准的章节名
    "供应商资格要求",
    "供应商条件",
    "申请人资格要求",
    
    # ── 竞争性谈判/询价特有 ──
    "供应商资格",
    "供应商资质",
    
    # ── 混合章节 ──
    "商务要求",
    "技术服务要求",
]
```

**注意**：新增关键词都是"章节标题"级别的，不是内容关键词。`_find_qualification_sections()` 原本的设计就是通过标题定位，这些新增项与其设计一致。

### 2. `_find_scoring_section()` 关键词扩展

```python
targets = [
    # ── 通用 ──
    "评分", "评分标准",
    "评审", "评审办法", "评审方法", "评审因素",
    
    # ── 公开招标特有 ──
    "评标办法", "综合评分",
    
    # ── 比选特有 ──
    "比选办法",
]
```

### 3. 评分表的混合章节检测增强

在 `_find_scoring_section()` 中，对标题含"评分"/"评审"的章节，增强内容表格检测：

```python
SCORE_TABLE_COLUMNS = ["评分因素", "评分项", "评审因素", "分值", "分数", "权重", "评分标准"]
```

如果章节标题匹配弱（如只含"评审"），但内容包含含上述列名的表格，则加分提升优先级。

## 验收标准
1. 符合规范的公开招标文件 → 正确定位"投标人资格""投标人须知"章节
2. 符合规范的比例文件 → 正确定位"供应商资格条件要求""比选须知"章节
3. 无资格章节的文档（极少数情况）→ 返回空列表（保持现有 fallback 行为）
4. 评分表检测不影响无评分内容的章节（如"合同模板"不应被误判为评分章节）

## 不包含
- 不改动章节搜索的评分算法（`_section_score()` 保持原逻辑）
- 不增加新的搜索维度（仍是标题 + 内容两阶段）
- 不改变 `scan_eligibility()` 的业务逻辑
- 不包含任何项目/招标文件特有关键词
