# 法规固定检查清单

## 目标
移除 `ELIGIBILITY_TEMPLATES` 矩阵，将法律法规硬性要求的检查项移至独立的 YAML 配置文件。

## 具体设计

### 配置位置
`config/presets/statutory_checklist.yaml`

### 配置内容
仅包含《政府采购法》第二十二条及其他法规硬性要求的通用资格条件：
- 独立承担民事责任能力
- 财务状况/财务报告
- 纳税和社保记录
- 信用记录（信用中国、政府采购网）
- 无重大违法记录声明
- 法定代表人授权书
- 行贿犯罪记录查询

### 清单结构
每个检查项包含：id、category、requirement、law_ref、severity

### 加载方式
启动时加载，缓存到内存。支持热加载（可选）。

### 匹配方式
在文档全文中搜索每项的关关键词，验证是否提及。
未在文档中提及的 stat 项标记为 "attention"，提示用户人工确认。

## 验收标准
1. 配置文件包含至少 8 项法规固定检查项
2. 每项有法律条文引用
3. 配置不依赖任何 bid_type 或 doc_type
4. API 返回的资格清单包含 statutory_items 标记
