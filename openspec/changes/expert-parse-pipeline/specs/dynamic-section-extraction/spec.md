# 动态章节提取

## 目标
强化章节定位机制，从文档中动态提取资格要求，不依赖 bid_type 和 doc_type 预设。

## 具体设计

### 章节定位 v2
在 `_find_qualification_sections()` 基础上引入评分机制：
- 对每个章节计算"资格相关性得分"
- 标题信号词加权（高置信度≥8分、中置信度5-7分、低置信度2-3分）
- 干扰章节扣分（目录、合同模板等）
- 内容关键密度加分（营业执照、资格、资质等出现次数）
- 取最高分章节的 80% 阈值作为资格章节集

### 评分信号词
不依赖硬编码的关键词列表，而是使用评分权重表：
```python
title_signals = [
    ("资格要求", 10),
    ("供应商资格", 10),
    ("投标人须知", 7),
    ("第四章", 5),
    ...
]
noise_signals = ["目录", "合同模板", "响应文件格式", ...]
density_signals = ["营业执照", "资格", "资质", "许可证", ...]
```

### 动态提取
从定位到的资格章节中：
1. 按行分割文本
2. 检测每行的信号词
3. 自动归类（用 signal_words.yaml）
4. 检测废标条件和 ★ 条款
5. 去重

### 无资格章节时
fallback：在全部文本中搜索废标条件和 ★ 条款。

## 验收标准
1. 对 10 份测试标书，资格章节定位准确率 ≥ 90%
2. 不需要 doc_type 参数
3. 不需要 bid_type 参数
4. 不需要 ELIGIBILITY_TEMPLATES
