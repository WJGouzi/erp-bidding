# 元数据提取 — 规格说明

## 所属层级：Phase 1

## 规则部分（保持现有）

### 字段：项目编号、项目名称

**输入**：文档前50个段落文本

**规则**：
```python
("project_code", r"采购项目编号[：:]\s*(\S+)")
("project_name", r"采购项目名称[：:]\s*(.+?)$")
```

**输出**：`metadata.project_code`, `metadata.project_name`

**覆盖率**：10/10 ✅ 全部可用

---

### 字段：分包结构

**输入**：全文

**规则修复**：
```python
# 新增: "采购包X" 格式（最常见）
("package_count", r"采购包(\d+)")

# 保留原有
("package_count", r"本项目共计\s*(\d+)\s*个包")
("package_count", r"第(\d+)包")
```

**逻辑**：取以上所有匹配的最大值。如果全部没匹配到，默认1。

**输出**：`metadata.package_count`, `metadata.packages[]`

**覆盖率**：当前 50% → 修复后 90%

---

### 字段：文档类型

**输入**：文档标题/第一页

**规则**：
```python
("bid_type", r"(公开招标|比选|竞争性谈判|竞争性磋商|询价)")
```

**问题**：同一文档可能含多个关键词（"公开招标"+"竞争性磋商"同时出现）

**修复**：取第一个出现的（文档封面最准确）

**覆盖率**：当前 80% → 修复后 90%

---

## LLM 部分（新增）

### 字段：购买人/代理机构名称

**输入**：文档前 3000 字符（含封面页）

**Prompt**：
```
你是一个招标文件解析专家。
从以下文档开头提取采购人（招标人）和采购代理机构的全称。

文档片段：
{document_excerpt}

以 JSON 格式返回：
{
  "purchaser_name": "采购人全称",
  "purchaser_contact": "联系人姓名（如无则null）",
  "agent_name": "代理机构全称",
  "agent_contact": "联系人姓名（如无则null）"
}
```

**输入示例**：
```
资阳市疾病预防控制中心
四川国信恒通项目管理有限公司共同编制
```

**期望输出**：
```json
{
  "purchaser_name": "资阳市疾病预防控制中心",
  "agent_name": "四川国信恒通项目管理有限公司",
  "purchaser_contact": null,
  "agent_contact": null
}
```

**覆盖率**：当前 70% → 修复后 100%

**兜底**：如果 LLM 调用失败，使用现有正则（覆盖70%+）

---

### 字段：预算 + 分包分配

**输入**：前附表KV对 + 文档前 5000 字符

**Prompt**：
```
从前附表中提取预算信息，按包拆分。

前附表关键信息：
{table_kv}

以 JSON 格式返回：
{
  "budget_total": 总金额（数字）,
  "budget_note": "原始金额文本",
  "packages": [
    {"package_no": 1, "amount": 金额, "note": "描述"},
    {"package_no": 2, "amount": 金额, "note": "描述"}
  ]
}
```

**输入示例**：
```
采购预算及最高限价★ → 本项目各包采购预算金额如下：
采购包1：354,530.00元
采购包2：1,119,842.20元
```

**期望输出**：
```json
{
  "budget_total": 1474372.20,
  "budget_note": "本项目各包采购预算金额",
  "packages": [
    {"package_no": 1, "amount": 354530.00, "note": "采购包1"},
    {"package_no": 2, "amount": 1119842.20, "note": "采购包2"}
  ]
}
```

**覆盖率**：当前 0% → 修复后 90%
