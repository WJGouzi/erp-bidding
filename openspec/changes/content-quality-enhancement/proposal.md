# 标书内容质量管理 — Proposal

> 更新日期: 2026-07-02

---

## 问题总结

### 1. 数据来源不分层
```
主体表已有字段 → 混入 LLM prompt
↓
确定性的公司名/信用代码被 LLM 模糊化处理
↓
应该直接填空，不走 LLM
```

### 2. 资格项没有递进查找
```
资格证明文件 → 只查主体材料 → 没有 → 空
↓
缺少"主体→知识库→留白"三级递进
↓
30项资格要求全部 MISSING
```

### 3. 产品列表断裂
```
招标文件的报价一览表 → 分析阶段已解析 → 生成阶段字段名不匹配
↓
产品数据在两个系统间丢失
↓
报价表内容空白
```

### 4. 承诺函降级风险
```
模板填充失败 → 降级到 LLM 生成 → 改写原文
↓
承诺函/声明函等固定格式被修改
↓
废标风险
```

### 5. 占位符识别方式
```
正则 + LLM 双兜底 → 正则无法穷尽所有格式
↓
新的占位符格式需要不断加正则
↓
应完全依赖 LLM 泛化识别
```

---

## 改造范围

| 模块 | 改动 | 难度 | 影响面 |
|------|------|------|--------|
| 资格引擎 | 三级递进查找 + 分页符 | 中 | 资格证明章节 |
| 表格引擎 | 原样复制 + 产品库匹配 | 中 | 报价一览表 |
| 填空引擎 | 纯LLM + 不移除降级 | 小 | 承诺函章节 |
| 字段映射 | 统一 PRODUCT_COLUMN_MAP | 小 | 所有表格处理 |
| 覆盖检查 | QUALIFICATION 特殊处理 | 小 | MISSING 计数 |

---

## 风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| LLM 占位符识别不可靠 | 保留正则位置修正，不做兜底提取 |
| 产品库匹配不准 | 匹配不到保持空白，不填充错误信息 |
| 表格样式不一致 | 原样复制招标文件的表头和行结构 |
| 分页符打乱文档布局 | 仅对 QUALIFICATION 类型应用，不影响其他章节 |
