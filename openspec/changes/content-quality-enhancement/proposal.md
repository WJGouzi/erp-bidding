# 标书内容质量管理 — Proposal

## Why：当前系统在三个层面有质量隐患

### 问题 1：数据来源不分层
```
主体表已有字段（法人、信用代码、公司名）→ 混入 LLM prompt 做"参考"
可能被 LLM 忽略或替换为低相关度知识库片段
确定性信息被模糊化处理
```

### 问题 2：固定格式内容走 LLM 生成而非填空
```
承诺函、声明函、授权书等 → LLM "写"而非填空
风险：改写原文、添加未声明承诺、遗漏关键表述
招标文件第三章"比选申请文件格式"是强制性规范
真实文件包含 9 种模板（文本填空+表格填写）
```

### 问题 3：资格证明文件被当作"写作任务"
```
第四章"资格证明文件"列出需提交的文档清单
（营业执照复印件、法定代表人身份证明、授权书……）
当前系统：→ LLM 生成一段文字描述
应该：→ 直接插入已上传的扫描件/图片
```

### 数据佐证
对 10 份真实招标文件的分析显示：
- 占位符格式超过 6 种变体，正则无法穷尽
- 第三章模板含 9 种格式：4 个文本填空 + 5 个表格
- 资格证明文件清单平均 5-8 项

## What：四路分流架构

```
输入章节
  │
  ├── 承诺函/声明函等文本模板 ──→ 填空引擎（LLM识别占位符 + 确定性替换）
  │
  ├── 报价表/应答表等表格模板 ──→ 表格填充引擎（生成 docx 真表格）
  │
  ├── 资格证明文件要求 ──→ 资料插入引擎（直接插扫描件/图片）
  │
  └── 方案/描述类 ──→ LLM 写作引擎（知识库 + 主体资料 + 置信度门控）
```

### 维度 A：填空引擎
- LLM 只做"识别占位符"（不碰原文）
- 字段映射表做确定性替换
- 替换前后 diff 校验

### 维度 B：表格填充引擎
- 检测表格类模板（报价表、偏离表、业绩表等）
- 生成 docx 真表格（`<w:tbl>` 结构）
- 从招标需求中提取数据填充

### 维度 C：资格证明文件插入引擎
- 从分析结果中提取"需提交的资格证明文件清单"
- 对照主体资料库检查是否已上传
- 已上传 → 直接插入扫描件/图片页
- 未上传 → 标记到"待人工补齐清单"

### 维度 D：置信度门控（后续）
- OCR 置信度、召回相关性、格式校验
- 低置信度数据不入 prompt

## How：实施步骤

### Step 1：填空引擎改造（P0）
- 用 LLM 识别替代正则识别占位符
- 保持确定性替换逻辑

### Step 2：表格填充引擎（P1）
- 表格模板检测
- docx 真表格生成

### Step 3：资格证明文件插入（P1）
- 要求清单提取
- 资料匹配与插入

### Step 4：置信度门控（P2）
- OCR 置信度、召回门控、格式校验

## 风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| LLM 识别占位符不够准 | 中 | 中 | 识别 + 正则双保险，低置信走 LLM 生成 |
| 表格结构多样 | 高 | 中 | 先覆盖常见表格（报价表、偏离表） |
| 资格证明文件匹配不准 | 中 | 低 | 匹配不上就走"待人工补齐" |
