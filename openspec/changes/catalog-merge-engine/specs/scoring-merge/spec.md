# 阶段2：评分合并 — 规格说明

## 输入

```python
scoring = {
    "dimensions": [
        {"name": "报价", "score": 30, "type": "objective"},
        {"name": "供应商业绩", "score": 10, "type": "objective"},
        {"name": "服务方案", "score": 40, "type": "subjective"},
        {"name": "售后保障方案", "score": 20, "type": "subjective"},
    ]
}
```

## 输出（修改后的 skeleton）

在原有骨架中插入新增章节。新增章节格式：

```python
{
    "source": "scoring",
    "source_dim": "服务方案",
    "title": "服务方案",
    "description": "根据本项目采购需求，编制服务方案",
    "children": [],
    "score": 40,
}
```

## 处理规则

| 维度类型 | 操作 | 说明 |
|---------|------|------|
| objective + 已覆盖 | 无操作 | 已在骨架中有对应章节 |
| objective + 未覆盖 | 警告日志 | 可能解析不完全，不阻塞 |
| subjective + 已覆盖 | 无操作 | 罕见情况 |
| subjective + 未覆盖 | 新增章节 | 插入到关键词最近匹配位置之后 |
| 合计/总计 | 跳过 | 汇总行 |

## 覆盖判定

### 显式映射表
```python
{
    "报价": ["报价一览表", "报价表", "报价部分"],
    "供应商业绩": ["类似项目业绩", "业绩一览表"],
    "业绩": ["类似项目业绩"],
}
```

### 隐式匹配
维度名包含于骨架章节标题 → 已覆盖

## 插入位置判定

1. 取维度名前2-3字搜索 skeleton 各节点 title
2. 所有匹配位置取最小 index+1（插入在该章节之后）
3. 无匹配时插入在"其他材料"之前

## 描述规则

章节 description 不写评分标准原文，只写"根据本项目采购需求，编制{维度名}"
