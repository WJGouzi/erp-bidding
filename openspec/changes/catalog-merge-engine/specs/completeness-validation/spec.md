# 阶段4：完整性验证 — 规格说明

## 输入

```python
outline = [...]  # 编号后的目录列表
document_chapters = ["第一章比选邀请", "第二章比选申请人须知", ...]
```

## 输出

```python
warnings = ["章节 '第二章比选申请人须知' 在目录中无明确对应"]
```

## 验证映射表

```python
CHAPTER_SECTION_MAP = [
    ("比选邀请", ["比选函"]),
    ("须知", ["比选函"]),
    ("申请文件格式", []),       # 格式要求=目录本身
    ("资格证明", ["资格证明"]),
    ("比选项目及要求", ["报价一览表", "商务", "技术", "偏离表"]),
    ("评选办法", ["服务方案", "售后保障", "评分"]),
    ("合同", []),              # 参考，不生成响应章节
]
```

## 处理规则

1. 对每个 document_chapter 检查是否有匹配的 outline 节点
2. 匹配方式：关键词（如"资格证明"）→ 预期节点标题关键词（如"资格"）
3. 预期为空列表的章节跳过检查（如"申请文件格式"、"合同"）
4. 跳过"目录"章节（文档自己的目录）
5. 只输出 warning，不阻塞生成

## 作用

- 提醒开发者：某章内容可能在目录中无对应
- 不是错误，不中断流程
- 可被下游用于 UI 提示
