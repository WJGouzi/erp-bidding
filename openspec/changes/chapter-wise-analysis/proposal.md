# 按章节逐项提取分析管线

## Why

当前 v3 管线使用三层层级结构（元数据→生死线→评分），
但实际标书撰写专家是按**章节顺序**阅读和提取信息的。
三层抽象与文档的实际组织方式之间存在语义鸿沟，导致：
- 多个信息点散落在不同章节但被忽略（开标时间、标书购买时间、服务期限等）
- 同类型信息在不同章节出现但缺乏关联（如包的中标人数量在须知前附表，包名在第一章）
- 章节特有的提取逻辑无法复用（须知前附表是表格，技术要求是文本，评审方法是评分表）

## 核心设计

以**章节为提取单元**，每个章节配置独立的提取器（extractor），
管线按章节顺序调度，最终合并为结构化输出。

```
文档
  │
  ├─ 第一章 提取器 ──→ 项目概况块
  ├─ 第二章 提取器 ──→ 须知前附表块（表格解析）
  ├─ 第三章 提取器 ──→ 资格条件块（已有）
  ├─ 第四章 提取器 ──→ 证明文件清单
  ├─ 第五章 提取器 ──→ 技术/服务要求块
  ├─ 第六章 提取器 ──→ 文件格式清单
  ├─ 第七章 提取器 ──→ 评分表（已有）
  └─ 第八章 提取器 ──→ 合同要点
       │
       ▼
   合并器 → 最终 analysis_data
```
